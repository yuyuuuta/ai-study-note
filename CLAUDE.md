# AI学習ノート サイト制作プロジェクト

## このプロジェクトの目的
生成AIパスポート試験の学習内容を、自分専用の学習サイトとしてまとめる。
「本質を理解する」ことを重視し、ユーザー自身の気づきやたとえを織り込むのが最大の特徴。
最終的にGitHub Pagesで一般公開する。

## ユーザーについて
- 生成AIパスポートを2026年6月最終週土曜に受験予定
- 暗記より本質理解を重視。背景・経緯・他分野との接続を大事にする
- 自分で見つけたたとえ（飛行機雲の「あ」など）で深く理解するタイプ

## サイトの構成
```
ai-study-site/
├── index.html          # トップページ
├── chapter1.html       # 1章目次（AIの基礎・5記事）
├── chapter2.html       # 2章目次（生成AIアーキテクチャ・5記事）
├── c1-01〜c1-05.html    # 1章の各記事
├── c2-01〜c2-05.html    # 2章の各記事
├── style.css           # 全ページ共通デザイン
├── script.js           # インタラクティブ動作
└── assets/             # キャラクター画像・図SVG
```

## デザイン方針（重要・厳守）
参考にしたサイト: https://ichi-hairsalon.com/ （上品・ミニマル・余白重視）
- クリーム背景(#faf8f5)、金アクセント(#c9a14a)、明朝体見出し
- すでに style.css に全スタイル定義済み。新規記事はこのCSSのクラスを使う
- 主要クラス：.article / .article-body / .toc（目次）/ .analogy（青・たとえ）/
  .insight-box（ピンク・あなたの気づき）/ .point（金・ポイント）/ .exam-box（試験ポイント）/
  .chara-block（キャラ吹き出し）/ .mini-card（カード）/ .cmp-table（比較表）/ .timeline

## 記事の書き方テンプレート（c1-02-shikumi.html が完成見本）
各記事は以下を必ず含める：
1. article-header（カテゴリ・タイトル・サブタイトル）
2. toc（記事内目次）
3. 本文（h2見出しで章立て、1記事3000〜4000字相当）
4. analogy（たとえボックス）を要所に
5. insight-box（ユーザーの気づき）を関連箇所に ← このサイトの肝
6. point（試験ポイント・注意点）
7. SVG図解を最低1つ（複雑な概念は図で見せる）
8. exam-box（記事末の試験ポイントまとめ）
9. chara-block（ユピテル先生の吹き出し）で締め

## ユーザーの重要な「気づき」一覧（insight-boxに使う）
- 「重み」＝「この意見は重みがある」の重み。どの入力を重要視するかの数字
- CNN：「森を見る前に木を見る感じ」「虫眼鏡で隅々を見て畳み込んで小さくする」
- Transformer：「重みの質が違う」（静的な重み→動的な重み）
- VAE：「游ゴシックも書道も飛行機雲の『あ』も生成できる。ふわっとすることで『あ』らしさを残しつつ創造できる」
- GAN：「バレーボールのスパイクとブロックの軍拡競争と同じ構造。競争が技術を加速させる」
- GPT：「GPTはプレ（Pre-trained）。Transformerで学習済みのものをTransformerで生成する」
- RNN前後：「記憶のない選手→短期記憶の選手→試合全体を覚える選手」
- 技術全般：「技術って改善の連続だな。できた！の直後に、でも○○が足りないが見つかる」
- 技術革新の3原則：「理論・計算・資源が揃ったとき爆発する」
